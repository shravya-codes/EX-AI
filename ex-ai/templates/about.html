Project Summary

Explainable AI-Driven User Behavior Threat Detection

Final year BCA project — Department of Computer Applications.

Objective
  • Detect suspicious user behavior using Machine Learning.
  • Analyze login attempts, session duration, pages accessed, and failed logins.
  • Classify users as Normal or Suspicious using a Random Forest Classifier.
  • Generate deviation scores and assign Low / Medium / High risk levels.
  • Use Explainable AI to interpret why the model made each prediction.
  • Visualize threat patterns through heatmaps, scatter plots, and a live dashboard.
How It Works
  • 1. Preprocessing — clean data, check duplicates/missing values.
  • 2. Feature Scaling — StandardScaler normalizes the 4 behavioral features.
  • 3. Training — Random Forest (200 trees) learns Normal vs Suspicious patterns.
  • 4. Evaluation — confusion matrix, precision/recall/F1 on a held-out test set.
  • 5. Explainability — permutation importance shows which features drive each decision.
  • 6. Deployment — Flask web app serves live predictions through this dashboard.
Tech Stack
Python scikit-learn Random Forest Classifier pandas / NumPy Matplotlib / Seaborn Permutation Importance (Explainable AI) Flask HTML / CSS / JavaScript
Why Random Forest?

Random Forest builds many decision trees on random subsets of the data and averages their output, which reduces overfitting and handles the mixed-scale behavioral features (login counts, minutes, page counts) reliably without heavy tuning — making it a strong, explainable baseline for a first cybersecurity ML classifier.
